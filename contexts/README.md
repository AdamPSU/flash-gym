# Contexts index

This directory is the project knowledge base. Read it before proposing architecture or writing code.

## Files

- `project-overview.md`: main project memory, including event notes, product seed, serverless strategy, constraints, and open questions.
- `runpod-flash.md`: verified Runpod Flash facts from documentation.
- `phase-1-keyframes.md`: phase 1 resource, endpoint, manifest, and upload notes.
- `phase-2-image-edit-models.md`: image editing model research and fallback options for phase 2.
- `phase-3-sam3.md`: phase 3 SAM3 segmentation contract, endpoint, runtime, and validation notes.
- `sources.md`: sources used to build this context.

## Source priority

- Direct team notes from the hackathon organizers or project team.
- Runpod Flash documentation.
- Local decisions recorded in this repository.
- External model or library documentation.
- Assumptions, only when clearly marked.

## Update rules

- Keep frontmatter in every context file.
- Keep notes factual and short.
- Add open questions instead of guessing.
- Update `sources.md` when a new external source changes the project context.
- Preserve exact names from team notes until a verified source proves a correction.
